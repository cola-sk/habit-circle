## 一、项目是什么

**HabitCircle（习惯圈）** 是一款面向**小学生**的习惯养成 Flutter 应用。

目标用户：一个小圈子里的小学生（如同一个兴趣班、学而思班级、邻居家孩子）。  
核心驱动：孩子完成学习类任务 → 获得积分 → 积分用来"养西瓜" → 西瓜越来越大 = 习惯越来越好。

---

## 二、核心理念：为什么是"养西瓜"？

> **养西瓜 → 谐音 → 养习惯**

西瓜是这款 App 里的虚拟成长载体（类似宠物），它代表孩子正在培养的习惯：

- 刚种下 → **西瓜种子**（刚开始培养习惯）
- 努力浇水（完成任务）→ **西瓜苗 → 小西瓜 → 大西瓜**（习惯逐渐成型）
- 长期坚持 → 习惯根深蒂固，西瓜丰收

**推广运营阶段的钩子**：真实世界里，如果孩子坚持完成任务、成功"养大"了虚拟西瓜，可以真的获得一个真实的大西瓜作为奖励，形成线上线下联动的激励闭环。

## 三、核心功能模块

### 3.1 任务系统（Task）

孩子每天完成各类学习任务，按完成时长累积积分：

| 任务类型 | 示例 | 计分方式 |
|---------|------|---------|
| 自主阅读 | 读课外书 | 每 15 分钟 10 分 |
| 练琴 | 钢琴、小提琴等 | 每 15 分钟 15 分 |
| 英语绘本 | 阅读/听英文绘本 | 每 15 分钟 12 分 |
| 课程预习 | 提前预习明天的功课 | 每 15 分钟 10 分 |
| 完成作业 | 完成当天作业 | 完成即得 20 分 |
| 户外运动 | 跑步、打球等 | 每 15 分钟 8 分 |
| 自定义任务 | 家长自定义 | 可自设分值 |

任务流程：孩子点"开始" → 计时器运行 → 点"完成" → 自动计算积分 → 积分进入待喂食池。

### 3.2 西瓜（宠物）系统（Pet/Watermelon）

> **代码中沿用 `pet` 命名**，但产品/UI 层面展示为西瓜形象。

- 每个孩子有一个西瓜（Pet），初始为种子状态
- 西瓜有**等级 1–10**，累计总积分达到阈值即升级，外观随等级成长（种子 → 西瓜苗 → 小西瓜 → 大西瓜）
- 每日任务积分用于"喂食"（浇水），当天积分越高，西瓜状态越好
- **西瓜状态**（每日刷新）：

  ```
  😄 开心   (当天积分 ≥ 80)
  😊 正常   (60–79 分)
  😕 饿了   (30–59 分)
  😢 很饿   (1–29 分)
  💀 奄奄一息 (0 分，且连续 2 天)
  ```

- 连续 7 天不喂食：状态重置（不彻底死亡，保留等级，鼓励重新开始）

### 3.3 圈子系统（Circle）

- 家长创建圈子，获得 6 位邀请码 / 二维码
- 其他家长扫码加入，同一孩子只属于一个圈子
- **圈子广场**：展示圈子内所有西瓜卡片（网格），实时更新，可以给别人的西瓜点"加油"
- 积分榜：今日圈子内前 3 名小朋友高亮展示，形成正向同伴压力

### 3.4 账号与认证

- **手机号 + 验证码**登录，无需密码（面向家长操作，简单易用）
- 首次登录：填写孩子昵称 → 给西瓜命名 → 创建或加入圈子

## 四、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 客户端 | Flutter 3 + Dart 3 | iOS & Android 跨平台 |
| 状态管理 | Riverpod 2 | `lib/providers/` |
| 路由 | GoRouter | `lib/core/router/` |
| 本地存储 | Hive | 离线任务记录、积分缓存 |
| 动画 | Rive | 西瓜/宠物状态动画（呼吸、开心、饥饿） |
| 后端 | Firebase (Firestore + Auth + FCM) | 实时数据库、手机号登录、Push 通知 |
| 后端补充 | Next.js (server/) | 服务端 API 补充，如复杂积分逻辑 |

## 六、Firestore 数据结构（关键集合）

```
/users/{uid}           - 用户信息（uid, phone, childName, circleId）
/pets/{uid}            - 西瓜数据（name, level, totalPoints, hungerStatus, lastFedAt）
/circles/{circleId}    - 圈子（name, inviteCode, memberUids[]）
/taskLogs/{logId}      - 任务记录（uid, taskType, durationMinutes, points, date）
/dailyFeeds/{uid_date} - 每日喂食记录（pointsEarned, pointsFed）
```

## 七、核心业务规则（写代码时需遵守）

1. **积分计算**：`points = floor(durationMinutes / 15) * pointsPerUnit`，单次任务积分上限由任务类型决定
2. **西瓜状态**：每天 00:00 重置，由 Cloud Function 计算；客户端根据 `lastFedAt` + 今日积分本地推算
3. **离线优先**：任务计时完全本地（Hive），联网后批量同步 Firestore
4. **圈子隔离**：用户只能查看自己所在圈子的数据；Firestore Security Rules 强制执行
5. **不收集敏感信息**：仅存储孩子昵称，不存储真实姓名、学校、班级等隐私数据
6. **适龄 UI**：大字体、明亮饱满配色、减少文字说明、操作路径不超过 3 步

## 八、设计风格

- **主色调**：绿色系（西瓜、自然、成长感）+ 西瓜红点缀
- **目标用户是小学生**：UI 要活泼、可爱、反馈感强（完成任务要有动画庆祝）
- **适龄字体**：正文 ≥ 16sp，按钮 ≥ 18sp
- **图标风格**：圆润、卡通，避免过于复杂的线条
